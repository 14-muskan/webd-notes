CSS SPECIFICITY

Factors that decide the style of the elements
  • Position and order of appearance: If any element is styled using multiple ways, then if they are styles using the same type of selector, the style that appears in the last is applied to the element.
  • Specifity:In all the cases where we have different types of selectors targeting the same element, either by element, or class or id, we calculate the specificity. Higher specificity means the style will be applied.
The order of the specificity is:     Inline Style > ID Selector > Class or Attribute Selector > Universal Selector(*) We should use a less specific selector (towards right) beacuse we can easily override general styles for any specific element.
  • Origin:The styles are either given in css or given by the browser. Inacse there is no css, browser styles are applied but preferenece is given to the css given by us. Browser's stylesheet is called user agent stylesheet
  • Importance:If we use !important for any style, then it will be applied surely, it can override inline styles.

Specificity calculations

To calculate the specificity, assign points to each type of selector as follows:
  • Universal selector: 0
  • Element selector and pseudo selector: 1
  • Class selector and attribute selector: 10
  • ID selector: 100
  • Inline styles: 1000
    • We keep adding these points as the selector involves any of these types. For ex: if the selector is h1.red.yellow[data-x] then the specificity will be 1(h1-element) + 10(red-class) + 10(yellow-class) + 10(data-x - attribute) = 31

    CSS SIZES

    • Pixels: It is denoted as px. It is NOT responsive
    • Viewport: It is denoted as vh(viewport height) and vw(viewport width)
    • Inherited size: denoted by em and rem. They work based on the size inherited by the element from its parent.
      • Em this is the multplying factor, by specifying the em value, we tell how much of the inherited value is to be given. Ex: if it is 2em and the inherited size is 16px, then the size would become 16*2 = 36px, helps in responsive design
      • Rem this is similar to em but this time we take the base size of the root element which is HTML
    • vmin and vmax For laptops, vmax = width and for mobile phones vmax = height. The size is interpreted as follows:
      If we specify 100vmin, then if the user is on laptop it will be 100% of the vmin on laptop which is width.
    • Min-height and Max-height Makes sure that the element occupies atleast the specified height irrespective of whether it contains that much content or not.
    • Percentage values they denote how much percent of the parent size should be given to the element

    CSS SHADOWS AND OUTLINES

  • Shadow : The syntax for box shadow is
    box-shadow: h-offset v-offset blur spread color inset
    • h-offset and v-offset : determines the shadow's horizontal and vertical position
    • blur : the larger the value, the blurrier the shadow
    • spread : expands or shrinks the shadow size
    • color : determines the color of the shadow
    • inset : makes shadow inner
    we can have shadows for both box and text with similar syntax, need not remeber the syntax, can always use inspect element on the developer's tools
  • Outline : different from border as it is not a part of the box model. It has various properties like offset that do NOT affect the box model of an element, border can vary on different sides but outline has uniform width on all sides.

    CSS OVERFLOW PROPERTY

    When the content overflows the box model, we can handle the overflow. overflow: scroll/auto/hidden/clip;
  • scroll means give the scroll bars always
  • auto means give the scroll bars if needed
  • hidden means the overflow is hidden, if it is used along with text-overflow:ellipsis; the overflow is denoted as ...
  • clip means we cannot scroll programmatically
    • we can also target the overflow in both dirctions as overflow-x and overflow-y. The shorthand is overflow: auto hidden; (first for x, then for y)
